# Muzukuru - Full-Stack To-Do Application

A complete full-stack to-do application with a Python Flask REST API backend and a React + TypeScript frontend. Features user authentication, CRUD operations, protected routes, and auto-generated API documentation.

## Tech Stack

### Backend
- **Framework:** Flask 3.1.3
- **Database:** SQLAlchemy 2.x + Flask-SQLAlchemy 3.x
- **API:** flask-smorest (OpenAPI 3.0.2)
- **Migrations:** Flask-Migrate / Alembic
- **Authentication:** PyJWT 2.12.1 + werkzeug security
- **Logging:** File + Email (SMTPHandler)
- **CORS:** flask-cors

### Frontend
- **Framework:** React 18+
- **Language:** TypeScript (strict mode)
- **Styling:** Tailwind CSS
- **HTTP Client:** Fetch API with custom service
- **Routing:** React Router v6+
- **State Management:** React Context API

## Setup

### Prerequisites
- Python 3.8+, just check the rest in requirements.txt
- Virtual environment (venv)

### Installation

```bash
# Navigate to backend directory
cd backend

# Create virtual environment
python -m venv venv

# Activate venv
# Windows
venv\Scripts\activate
# Linux/Mac
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Initialize database
flask db upgrade
```

### Run Development Server

```bash
flask run
```

Server runs at `http://127.0.0.1:5000`

## API Endpoints

### Health Check
- `GET /health` — Status check

### Swagger UI
- `GET /docs` — OpenAPI documentation (Swagger UI)
- `GET /openapi.json` — Raw OpenAPI spec

### Authentication
- `POST /api/auth/register` — Create account with username, email, password → returns JWT
- `POST /api/auth/login` — Login with username, password → returns JWT

### Users (CRUD) — 🔒 All Protected (requires Bearer token)
- `GET /api/users/` — List all users
- `POST /api/users/` — Create user
- `GET /api/users/{user_id}` — Get single user
- `PUT /api/users/{user_id}` — Update user
- `DELETE /api/users/{user_id}` — Delete user

### Todos (CRUD) — 🔒 All Protected (requires Bearer token)
- `GET /api/todos` — List user's todos (sorted by newest first)
- `POST /api/todos` — Create new todo with title and optional description
- `GET /api/todos/{todo_id}` — Get specific todo (only if owner)
- `PUT /api/todos/{todo_id}` — Update todo (title, description, completed status)
- `DELETE /api/todos/{todo_id}` — Delete todo (only if owner)

## Error Handling

All errors return JSON with `error` and `message` fields:

```json
{
  "error": "Not Found",
  "message": "The requested resource was not found."
}
```

Status codes:
- `400` — Bad Request
- `401` — Unauthorized (missing/invalid token)
- `404` — Not Found
- `405` — Method Not Allowed
- `409` — Conflict (unique constraint)
- `422` — Unprocessable Entity (validation)
- `500` — Internal Server Error

## JWT Authentication

### Register a User

```bash
curl -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "john_doe",
    "email": "john@example.com",
    "password": "securepassword123"
  }'
```

Response:
```json
{
  "access_token": "eyJ0eXAiOiJKV1QiLCJhbGc...",
  "token_type": "Bearer",
  "user": {
    "id": 1,
    "username": "john_doe",
    "email": "john@example.com",
    "is_active": true
  }
}
```

### Login

```bash
curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "john_doe",
    "password": "securepassword123"
  }'
```

### Access Protected Endpoints

Include the token in the `Authorization` header:

```bash
curl -X GET http://localhost:5000/api/users/ \
  -H "Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGc..."
```

## Logging

The application logs errors to both file and email (if configured).

### File Logging

All logs are written to `app.log` in the backend directory. The log file includes:
- Request errors (4xx, 5xx)
- Database errors
- Unhandled exceptions

### Email Logging (Optional)

To receive error notifications via email, configure SMTP credentials in `.flaskenv`:

**Gmail Example:**
```bash
MAIL_SERVER=smtp.gmail.com
MAIL_USERNAME=your-email@gmail.com
MAIL_PASSWORD=your-16-char-app-password  # Use app-specific password
ADMINS=admin@example.com,dev@example.com
```

**SendGrid Example:**
```bash
MAIL_SERVER=smtp.sendgrid.net
MAIL_USERNAME=apikey
MAIL_PASSWORD=SG.xxxxxxxxxxxxx
ADMINS=admin@example.com
```

When configured, errors automatically email admins with full traceback.
