# Muzukuru API - Flask Backend

A Python Flask REST API for a full-stack to-do application with user authentication, CRUD operations, and auto-generated API documentation.

## Tech Stack

- **Framework:** Flask 3.1.3
- **Database:** SQLAlchemy 2.x + Flask-SQLAlchemy 3.x
- **API:** flask-smorest (OpenAPI 3.0.2)
- **Migrations:** Flask-Migrate / Alembic
- **Authentication:** PyJWT 2.12.1 + werkzeug security
- **CORS:** flask-cors (todo)
- **Logging:** Python logging (todo)

## Setup

### Prerequisites
- Python 3.8+
- Virtual environment (venv)

### Installation

```bash
# Navigate to backend directory
cd backend

# Create virtual environment
python -m venv venv

# Activate venv
# Windows
venv\Scripts\activate
# Linux/Mac
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Initialize database
flask db upgrade
```

### Run Development Server

```bash
flask run
```

Server runs at `http://127.0.0.1:5000`

## API Endpoints

### Health Check
- `GET /health` вЂ” Status check

### Swagger UI
- `GET /docs` вЂ” OpenAPI documentation (Swagger UI)
- `GET /openapi.json` вЂ” Raw OpenAPI spec

### Users (CRUD)
- `GET /api/users/` вЂ” List all users
- `POST /api/users/` вЂ” Create user
- `GET /api/users/{user_id}` вЂ” Get single user
- `PUT /api/users/{user_id}` вЂ” Update user
- `DELETE /api/users/{user_id}` вЂ” Delete user

## Environment Variables

Create `.flaskenv` or set environment variables:

```bash
FLASK_ENV=development
FLASK_DEBUG=1
DATABASE_URL=sqlite:///development.db
SECRET_KEY=your-secret-key-here
```

## Database Migrations

```bash
# Create migration (after model changes)
flask db migrate -m "Description of changes"

# Apply migrations
flask db upgrade

# View history
flask db history
```

## Error Handling

All errors return JSON with `error` and `message` fields:

```json
{
  "error": "Not Found",
  "message": "The requested resource was not found."
}
```

Status codes:
- `400` вЂ” Bad Request
- `404` вЂ” Not Found
- `405` вЂ” Method Not Allowed
- `409` вЂ” Conflict (unique constraint)
- `422` вЂ” Unprocessable Entity (validation)
- `500` вЂ” Internal Server Error
