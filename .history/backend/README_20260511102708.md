# Muzukuru API - Flask Backend

A Python Flask REST API for a full-stack to-do application with user authentication, CRUD operations, and auto-generated API documentation.

## Tech Stack

- **Framework:** Flask 3.1.3
- **Database:** SQLAlchemy 2.x + Flask-SQLAlchemy 3.x
- **API:** flask-smorest (OpenAPI 3.0.2)
- **Migrations:** Flask-Migrate / Alembic
- **Authentication:** PyJWT 2.12.1 + werkzeug security
- **CORS:** flask-cors (todo)
- **Logging:** Python logging (todo)

## Setup

### Prerequisites
- Python 3.8+
- Virtual environment (venv)

### Installation

```bash
# Navigate to backend directory
cd backend

# Create virtual environment
python -m venv venv

# Activate venv
# Windows
venv\Scripts\activate
# Linux/Mac
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Initialize database
flask db upgrade
```

### Run Development Server

```bash
flask run
```

Server runs at `http://127.0.0.1:5000`

## API Endpoints

### Health Check
- `GET /health` — Status check

### Swagger UI
- `GET /docs` — OpenAPI documentation (Swagger UI)
- `GET /openapi.json` — Raw OpenAPI spec

### Authentication
- `POST /api/auth/register` — Create account with username, email, password → returns JWT
- `POST /api/auth/login` — Login with username, password → returns JWT

### Users (CRUD) — 🔒 All Protected (requires Bearer token)
- `GET /api/users/` — List all users
- `POST /api/users/` — Create user
- `GET /api/users/{user_id}` — Get single user
- `PUT /api/users/{user_id}` — Update user
- `DELETE /api/users/{user_id}` — Delete user

## Error Handling

All errors return JSON with `error` and `message` fields:

```json
{
  "error": "Not Found",
  "message": "The requested resource was not found."
}
```

Status codes:
- `400` — Bad Request
- `401` — Unauthorized (missing/invalid token)
- `404` — Not Found
- `405` — Method Not Allowed
- `409` — Conflict (unique constraint)
- `422` — Unprocessable Entity (validation)
- `500` — Internal Server Error

## JWT Authentication

### Register a User

```bash
curl -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "john_doe",
    "email": "john@example.com",
    "password": "securepassword123"
  }'
```

Response:
```json
{
  "access_token": "eyJ0eXAiOiJKV1QiLCJhbGc...",
  "token_type": "Bearer",
  "user": {
    "id": 1,
    "username": "john_doe",
    "email": "john@example.com",
    "is_active": true
  }
}
```

### Login

```bash
curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "john_doe",
    "password": "securepassword123"
  }'
```

### Access Protected Endpoints

Include the token in the `Authorization` header:

```bash
curl -X GET http://localhost:5000/api/users/ \
  -H "Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGc..."
```
