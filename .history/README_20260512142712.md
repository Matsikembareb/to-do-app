# To-Do App

A full-stack to-do application with a React TypeScript frontend and Flask backend. Users can register, log in, and manage their todos with a clean, responsive interface.

## Features

- **User Authentication**: Register new accounts and log in with JWT tokens
- **Protected Dashboard**: Access restricted to authenticated users only
- **Todo Management**: Create, read, update, and delete todos
- **Responsive Design**: Works on desktop and mobile devices
- **Real-time UI Updates**: Loading states and error handling throughout

## Tech Stack

**Frontend:**
- React 19.2.6 with TypeScript (strict mode)
- Vite 8.0.12 for development
- React Router 7.15 for navigation
- Tailwind CSS 3.4.3 for styling
- Context API for state management

**Backend:**
- Flask 3.1.3
- SQLAlchemy 2.0.49 for database ORM
- PyJWT 2.12.1 for JWT authentication
- Flask-CORS 6.0.1 for cross-origin requests
- Alembic for database migrations

## Getting Started

### Prerequisites

- Python 3.8+
- Node.js 18+
- pip and npm installed

### Backend Setup

1. Navigate to the backend folder:
```bash
cd backend
```

2. Create a virtual environment:
```bash
python -m venv venv
```

3. Activate the virtual environment:
```bash
# Windows
venv\Scripts\activate
# macOS/Linux
source venv/bin/activate
```

4. Install dependencies:
```bash
pip install -r requirements.txt
```

5. Run the backend server:
```bash
python run.py
```

The backend will start on `http://localhost:5000`

### Frontend Setup

1. In a new terminal, navigate to the frontend folder:
```bash
cd frontend
```

2. Install dependencies:
```bash
npm install
```

3. Start the development server:
```bash
npm run dev
```

The frontend will be available at `http://localhost:3000`

## How It Works

### Authentication Flow

1. User registers with username, email, and password
2. Backend hashes the password and stores the user
3. User logs in with credentials
4. Backend validates credentials and returns a JWT token
5. Frontend stores the token in localStorage
6. All subsequent requests include the token in the Authorization header

### Protected Routes

The ProtectedRoute component checks if a user is authenticated before allowing access to protected pages. If not authenticated, it redirects to the login page.

### Todo Operations

The dashboard displays all todos for the logged-in user. Users can:
- Create new todos with a title and description
- View all their todos
- Mark todos as complete/incomplete
- Delete todos

All operations include loading states and error handling.

## Environment Variables

Create a `.env` file in the backend folder if needed for configuration (not included by default).

## Database

The application uses SQLite by default (stored in `instance/` folder). Database migrations are managed with Alembic. To apply migrations:

```bash
cd backend
alembic upgrade head
```
